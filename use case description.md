### Use Case: 회원 가입

* **Actor**: 방문자
* **사전 조건**: 시스템에 로그인되어 있지 않은 상태
* **후조건**: 새로운 회원 계정이 생성됨

**시나리오**:

1. 방문자는 회원 가입 버튼을 클릭한다.
2. 시스템은 회원 가입 입력 화면을 표시한다.
3. 방문자는 ID, 비밀번호, 전화번호, 결제 수단, 선호 자전거 유형을 입력한다.
4. 시스템은 입력값의 유효성을 검사한다.
5. 중복 ID가 없고 모든 필수 항목이 유효하면 회원 정보가 저장된다.
6. 회원 가입 완료 메시지가 표시된다.

### Use Case: 대여소 조회

* **Actor**: 관리자
* **사전 조건**: 로그인 상태
* **후조건**: 대여소 리스트가 출력됨

**시나리오**:

1. 관리자는 대여소 조회 메뉴를 클릭한다.
2. 시스템은 대여소 리스트를 출력한다.
3. 관리자는 필터링 혹은 특정 대여소를 선택할 수 있다.

---

### Use Case: 대여소 상세정보 확인

* **Actor**: 관리자
* **사전 조건**: 대여소 리스트에서 특정 항목이 선택됨
* **후조건**: 상세정보 화면이 출력됨

**시나리오**:

1. 관리자는 대여소 조회 결과 화면에서 특정 대여소를 클릭한다.
2. 시스템은 해당 대여소의 상세 정보를 출력한다.

---

### Use Case: 대여소 삭제

* **Actor**: 관리자
* **사전 조건**: 대여소 리스트 조회 후 특정 대여소 선택됨
* **후조건**: 해당 대여소가 삭제됨

**시나리오**:

1. 관리자는 대여소 리스트에서 삭제할 대여소를 선택한다.
2. 시스템은 삭제 여부를 확인하는 팝업을 표시한다.
3. 관리자는 삭제를 확인한다.
4. 문제가 없으면 해당 대여소를 삭제하고 완료 메시지를 출력한다.

### Use Case: 자전거 등록

* **Actor**: 관리자
* **사전 조건**: 
* **후조건**: 새로운 자전거가 등록된 자전거들의 리스트에 추가됨

**시나리오**:

1. 자전거 등록에 필요한 항목의 정보를 입력할 빈칸을 화면에 출력한다.
2. 액터가 자전거 등록에 필요한 항목의 정보를 주어진 빈칸에 맞게 작성한다.
3. 작성된 정보를 나열하여 확인하는 팝업을 화면에 출력한다.
4. 작성된 정보가 맞으면 확인, 수정이 필요하면 취소를 누른다.
5. 등록이 완료되었다는 메시지를 출력한다.

### Use Case: 자전거 삭제

* **Actor**: 관리자
* **사전 조건**: 
* **후조건**: 

**시나리오**:

1. 등록된 자전거 리스트를 출력한다.
2. 리스트에 있는 항목 중 삭제하고 싶은 항목을 선택한다.
