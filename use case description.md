## 📄 Use Case Descriptions (Step-by-Step Breakdown)

---

### Use Case: 회원 가입

* **Actor**: 방문자

1. 시스템은 메인 화면에 '회원 가입' 버튼을 표시한다.
2. 방문자는 '회원 가입' 버튼을 클릭한다.
3. 시스템은 회원 가입 입력 화면을 표시한다.
4. 방문자는 ID, 비밀번호, 전화번호, 결제 수단, 선호 자전거 유형(일반/전기) 등을 입력하고 ‘가입 완료’ 버튼을 클릭한다.
5. 시스템은 회원 가입이 완료되었음을 알리는 메시지를 화면에 표시한다.

---

### Use Case: 대여소 등록

* **Actor**: 관리자

1. 시스템은 메인 화면에 '대여소 등록' 버튼을 표시한다.
2. 관리자는 '대여소 등록' 버튼을 클릭한다.
3. 시스템은 대여소 등록 입력 화면을 표시한다.
4. 관리자는 대여소 이름, 위치, 보관 가능 수량, 운영 시간 등을 입력하고 '등록 완료' 버튼을 클릭한다.
5. 시스템은 대여소 등록이 완료되었음을 알리는 메시지를 화면에 표시한다.

---

### Use Case: 대여소 리스트 조회

* **Actor**: 관리자

1. 시스템은 메인 화면에 '대여소 조회' 버튼을 화면에 표시한다.
2. 관리자는 '대여소 조회' 버튼을 클릭한다.
3. 시스템은 등록된 대여소 목록을 표시하는 ‘대여소 리스트 조회 화면’을 출력한다.

---

### Use Case: 대여소 상세정보 조회

* **Actor**: 관리자

1. 시스템은 메인 화면에 '대여소 조회' 버튼을 화면에 표시한다.
2. 관리자는 '대여소 조회' 버튼을 클릭한다.
3. 시스템은 등록된 대여소 목록을 표시하는 ‘대여소 리스트 조회 화면’을 출력한다.
4. 관리자는 조회된 목록에서 특정 대여소를 선택한다.
5. 시스템은 선택한 대여소의 상세 정보를 화면에 표시한다.

---

### Use Case: 대여소 삭제

* **Actor**: 관리자

1. 시스템은 메인 화면에 '대여소 조회' 버튼을 화면에 표시한다.
2. 관리자는 '대여소 조회' 버튼을 클릭한다.
3. 시스템은 등록된 대여소 목록을 표시하는 ‘대여소 리스트 조회 화면’을 출력한다.
4. 관리자는 목록에서 삭제할 대여소를 선택한다.
5. 시스템은 삭제 여부를 확인하는 팝업을 표시한다.
6. 관리자는 삭제를 확인한다.
7. 시스템은 선택한 대여소를 삭제하고, 삭제 완료 메시지를 표시한다.

---

### Use Case: 자전거 등록

* **Actor**: 관리자

1. 시스템은 등록 입력칸을 출력한다.  
2. 관리자는 자전거 정보를 입력한다.  
3. 시스템은 확인 팝업을 출력한다.  
4. 관리자는 확인을 누르면 등록을 진행한다.  
5. 시스템이 등록 완료 메시지를 출력한다.

---

### Use Case: 자전거 리스트 조회

* **Actor**: 관리자

1. 시스템은 등록된 자전거 리스트를 출력한다.

---

### Use Case: 자전거 삭제

* **Actor**: 관리자

1. 시스템은 자전거 리스트를 출력한다.  
2. 관리자는 삭제할 항목을 선택한다.  
3. 시스템은 간단한 정보 요약을 통해 재확인을 유도한다.  
4. 관리자는 확인 버튼을 누른다.  
5. 시스템은 삭제를 수행한 후 갱신된 리스트를 출력한다.

---

### Use Case: 자전거 상세 정보 조회

* **Actor**: 관리자

1. 시스템이 자전거 리스트를 출력한다.  
2. 관리자는 특정 자전거 항목을 선택한다.  
3. 시스템은 상세 정보를 출력한다.

---

### Use Case: 대여 기록 최근순 조회

* **Actor**: 관리자

1. 시스템은 자전거 대여 정보를 반납 시간 기준으로 정렬하여 최근순으로 출력한다.

---

### Use Case: 대여 기록 지역별 조회

* **Actor**: 관리자

1. 시스템은 대여 기록을 출력한다.  
2. 관리자는 지역 기준 정렬 옵션을 선택한다.  
3. 시스템은 지역 기준으로 정렬된 데이터를 출력한다.

---

### Use Case: 대여 금액 및 횟수 통계 조회

* **Actor**: 관리자

1. 시스템은 전체 대여 금액 및 횟수 정보를 통계 형태로 출력한다.

---

### Use Case: 대여 금액 및 횟수 통계 기간별 조회

* **Actor**: 관리자

1. 시스템은 관리자에게 기간별 조회(1주, 1개월, 1년) 옵션을 제공한다.  
2. 관리자는 조회하고자 하는 기간을 선택한다.
3. 시스템은 선택된 기간에 따라 대여 금액과 횟수 통계를 출력한다.

---

### Use Case: 로그인

* **Actor**: 회원

1. 로그인 화면에서 ID와 비밀번호를 입력한다.  
2. 로그인 버튼을 클릭한다.  
3. 입력값이 유효하면 로그인을 완료한다.

---

### Use Case: 로그아웃

* **Actor**: 회원

1. 로그아웃 버튼을 클릭한다.  
2. 시스템이 로그아웃을 수행한다.

---

### Use Case: 회원 탈퇴

* **Actor**: 회원

1. 회원 탈퇴 버튼을 클릭한다.  
2. 시스템이 탈퇴 여부를 묻는다.  
3. 사용자가 확인하면 탈퇴를 수행하고 관련 데이터를 삭제한다.

---

### Use Case: 대여소 검색

* **Actor**: 회원

1. 시스템이 검색창을 출력한다.  
2. 회원이 대여소 이름을 입력하여 검색한다.  
3. 조건에 맞는 대여소 리스트를 출력한다.

---

### Use Case: 대여소 상세 정보 확인

* **Actor**: 회원

1. 대여소 검색 결과 화면을 확인한다.  
2. 특정 대여소를 선택한다.  
3. 해당 대여소의 상세 정보를 출력한다.

---

### Use Case: 자전거 즉시 대여

* **Actor**: 회원

1. 대여소 상세 정보 화면에서 사용 가능한 자전거를 확인한다.  
2. 즉시 대여 버튼을 클릭한다.  
3. 대여 완료 메시지 및 문자 알림을 전송한다.

---

### Use Case: 자전거 예약 대기

* **Actor**: 회원

1. 대여소 상세 정보 화면에서 자전거가 없음을 확인한다.  
2. 예약 대기를 요청한다.  
3. 시스템이 예약 대기 접수를 완료하고 문자로 알림을 보낸다.

---

### Use Case: 대여 중인 자전거 정보 조회

* **Actor**: 회원

1. 시스템은 '대여 정보 조회' 버튼을 화면에 표시한다.
2. 회원은 '대여 정보 조회' 버튼을 클릭한다.  
3. 시스템은 현재 대여 중인 자전거 리스트를 출력한다.

---

### Use Case: 자전거 반납

* **Actor**: 회원

1. 시스템은 '대여 정보 조회' 버튼을 화면에 표시한다.
2. 회원은 '대여 정보 조회' 버튼을 클릭한다.
3. 시스템은 현재 대여 중인 자전거 리스트를 출력한다.
4. 회원은 '반납' 버튼을 클릭한다.
5. 시스템은 대여소 목록을 출력한다.
6. 회원은 대여소를 선택하고 반납을 확정한다.
7. 시스템은 반납 처리를 하고 요금을 자동 결제 한다.

---

### Use Case: 식당 예약

* **Actor**: 회원

1. 시스템은 '대여 정보 조회' 버튼을 화면에 표시한다.
2. 회원은 '대여 정보 조회' 버튼을 클릭한다.
3. 시스템은 현재 대여 중인 자전거 리스트를 출력한다.
4. 회원은 '반납' 버튼을 클릭한다.
5. 시스템은 대여소 목록을 출력한다.
6. 회원은 대여소를 선택하고 반납을 확정한다.
7. 시스템은 요금을 자동 결제 한 후 식당 예약 추천 팝업을 화면에 표시한다.
8. 회원은 원하는 경우 식당을 선택한다.
9. 시스템은 '예약 서비스로 이동합니다'를 출력한다.
