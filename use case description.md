## 📄 Use Case Descriptions (Step-by-Step Breakdown)

---

### Use Case: 회원 가입

* **Actor**: 방문자

1. 시스템은 메인 화면에 '회원 가입' 버튼을 표시한다.  
2. 방문자는 '회원 가입' 버튼을 클릭한다.  
3. 시스템은 회원 가입 입력 화면을 표시한다.  
4. 방문자는 ID, 비밀번호, 전화번호, 결제 수단, 선호 자전거 유형(일반/전기) 등을 입력하고 ‘가입 완료’ 버튼을 클릭한다.  
5. 시스템은 회원 가입이 완료되었음을 알리는 메시지를 화면에 표시한다.

---

### Use Case: 대여소 등록

* **Actor**: 관리자

1. 시스템은 메인 화면에 '대여소 등록' 버튼을 표시한다.  
2. 관리자는 '대여소 등록' 버튼을 클릭한다.  
3. 시스템은 대여소 등록 입력 화면을 표시한다.  
4. 관리자는 대여소 이름, 위치, 보관 가능 수량, 운영 시간 등을 입력하고 '등록 완료' 버튼을 클릭한다.  
5. 시스템은 대여소 등록이 완료되었음을 알리는 메시지를 화면에 표시한다.

---

### Use Case: 대여소 목록 조회

* **Actor**: 관리자

1. 시스템은 메인 화면에 '대여소 조회' 버튼을 표시한다.  
2. 관리자는 '대여소 조회' 버튼을 클릭한다.  
3. 시스템은 등록된 대여소 목록을 출력한다.

---

### Use Case: 대여소 상세 정보 조회

* **Actor**: 관리자

1. 관리자가 대여소 조회 화면을 연다.  
2. 등록된 대여소 목록이 출력된다.  
3. 관리자가 특정 대여소를 선택한다.  
4. 시스템이 상세 정보를 출력한다.

---

### Use Case: 대여소 삭제

* **Actor**: 관리자

1. 관리자가 대여소 목록 화면을 연다.  
2. 삭제할 대여소를 선택한다.  
3. 삭제 여부를 묻는 팝업이 표시된다.  
4. 관리자가 확인하면 해당 대여소를 삭제한다.  
5. 삭제 완료 메시지를 출력한다.

---

### Use Case: 자전거 등록

* **Actor**: 관리자

1. 등록 입력칸을 출력한다.  
2. 관리자가 자전거 정보를 입력한다.  
3. 확인 팝업이 표시된다.  
4. 확인을 누르면 등록을 진행한다.  
5. 등록 완료 메시지를 출력한다.

---

### Use Case: 자전거 목록 조회

* **Actor**: 관리자

1. 시스템은 등록된 자전거 목록을 출력한다.

---

### Use Case: 자전거 삭제

* **Actor**: 관리자

1. 자전거 목록을 출력한다.  
2. 삭제할 항목을 선택한다.  
3. 간단한 정보 요약을 통해 재확인을 유도한다.  
4. 확인 버튼을 누르면 삭제를 수행한다.  
5. 갱신된 목록을 출력한다.

---

### Use Case: 자전거 상세 정보 조회

* **Actor**: 관리자

1. 자전거 목록을 출력한다.  
2. 특정 자전거 항목을 선택한다.  
3. 상세 정보를 출력한다.

---

### Use Case: 대여 기록 최근순 조회

* **Actor**: 관리자

1. 자전거 대여 정보를 반납 시간 기준으로 정렬하여 최근순으로 출력한다.

---

### Use Case: 대여 기록 지역별 조회

* **Actor**: 관리자

1. 대여 기록을 출력한다.  
2. 지역 기준 정렬 옵션을 선택한다.  
3. 지역 기준으로 정렬된 데이터를 출력한다.

---

### Use Case: 대여 금액 및 횟수 통계 조회

* **Actor**: 관리자

1. 전체 대여 금액 및 횟수 정보를 통계 형태로 조회한다.

---

### Use Case: 대여 금액 및 횟수 기간별 통계 조회

* **Actor**: 관리자

1. 관리자에게 기간별 조회(1주, 1개월, 1년) 옵션을 제공한다.  
2. 선택한 기간에 따라 대여 금액과 횟수를 출력한다.

---

### Use Case: 로그인

* **Actor**: 회원

1. 로그인 화면에서 ID와 비밀번호를 입력한다.  
2. 로그인 버튼을 클릭한다.  
3. 입력값이 유효하면 로그인을 완료한다.

---

### Use Case: 로그아웃

* **Actor**: 회원

1. 로그아웃 버튼을 클릭한다.  
2. 시스템이 로그아웃을 수행한다.

---

### Use Case: 회원 탈퇴

* **Actor**: 회원

1. 회원 탈퇴 버튼을 클릭한다.  
2. 시스템이 탈퇴 여부를 묻는다.  
3. 사용자가 확인하면 탈퇴를 수행하고 관련 데이터를 삭제한다.

---

### Use Case: 대여소 검색

* **Actor**: 회원

1. 시스템이 검색창을 출력한다.  
2. 회원이 대여소 이름을 입력하여 검색한다.  
3. 조건에 맞는 대여소 리스트를 출력한다.

---

### Use Case: 대여소 상세 정보 확인

* **Actor**: 회원

1. 대여소 검색 결과 화면을 확인한다.  
2. 특정 대여소를 선택한다.  
3. 해당 대여소의 상세 정보를 출력한다.

---

### Use Case: 자전거 즉시 대여

* **Actor**: 회원

1. 대여소 상세 정보 화면에서 사용 가능한 자전거를 확인한다.  
2. 즉시 대여 버튼을 클릭한다.  
3. 대여 완료 메시지 및 문자 알림을 전송한다.

---

### Use Case: 자전거 예약 대기

* **Actor**: 회원

1. 대여소 상세 정보 화면에서 자전거가 없음을 확인한다.  
2. 예약 대기를 요청한다.  
3. 시스템이 예약 대기 접수를 완료하고 문자로 알림을 보낸다.

---

### Use Case: 대여 중 자전거 정보 조회

* **Actor**: 회원

1. 대여 중인 자전거 목록 조회 메뉴를 클릭한다.  
2. 현재 대여 중인 자전거 리스트를 출력한다.

---

### Use Case: 자전거 반납

* **Actor**: 회원

1. 대여 중인 자전거를 선택한다.  
2. 반납할 대여소를 선택하고 반납을 확정한다.  
3. 요금 자동 결제를 수행한다.  
4. 식당 추천에 동의하면 외부 서비스와 연동한다.

---

### Use Case: 식당 예약 연계

* **Actor**: 회원

1. 반납 후 근처 식당 추천을 수신한다.  
2. 사용자가 식당을 선택하면 예약 서비스로 연계된다.

---

### Use Case: 대여 기록 조회

* **Actor**: 회원

1. 과거 대여 기록 조회 메뉴를 클릭한다.  
2. 시스템은 날짜순으로 대여 기록을 출력한다.

---

### Use Case: 대여 기록 정렬 조회

* **Actor**: 회원

1. 대여 기록 조회 화면에서 정렬 옵션을 선택한다.  
2. 대여소 기준으로 정렬된 기록을 출력한다.

---

### Use Case: 대여 기록 삭제

* **Actor**: 회원

1. 삭제할 기록 항목을 선택한다.  
2. 삭제 확인 메시지를 받고 확인한다.  
3. 해당 항목을 삭제하고 목록에서 제거한다.

---

### Use Case: 예약 대기 목록 조회

* **Actor**: 회원

1. 예약 대기 정보 메뉴를 클릭한다.  
2. 예약 대기 자전거 목록을 출력한다.

---

### Use Case: 예약 대기 취소

* **Actor**: 회원

1. 예약 대기 목록에서 취소할 항목을 선택한다.  
2. 시스템이 취소 확인을 요청한다.  
3. 회원이 확인하면 예약이 취소된다.

---

### Use Case: 요금 및 사용 시간 조회

* **Actor**: 회원

1. 요금 조회 메뉴를 클릭한다.  
2. 대여 시간 및 요금 정보를 출력한다.
